App Hosting Platform

Course: Skills Lab Integration 2

Assignment:Created a hosting platform in a local datacenter to host a PHP application for a client.

Technology:

  • #VMware
  • #Linux

Team: Harold De Ridder, Tijs Clauwers, Lukas Mues, Kynan Van Looy, Quinten De Meyer, Jonas Speltinckx

Context & Background

The project involves the design and implementation of a hosting platform tailored for PHP/Laravel applications, hosted in the Thomas More datacenter. It supports automated deployment, focuses on scalability, security, and high availability, and enables clients to host multiple applications efficiently. The platform emphasizes control, cost-efficiency via shared hosting, and the ability to provide better support by removing reliance on third-party providers.

The platform's value proposition lies in its customizability, enhanced security, and full ownership of the infrastructure stack. However, it consciously excludes features like disaster recovery, SLA guarantees, 24/7 support, or multi-region deployments.

Realization

The platform was built with a strong focus on automation and layered complexity:

Crawl-Walk-Run-Fly Model:

  • Crawl: Manual tasks with basic automation.
  • Walk: Partial automation (e.g., using Ansible), basic monitoring.
  • Run: Full Infrastructure-as-Code, centralized logging.
  • Fly: Advanced features like auto-scaling and multi-tenancy.

Security Implementation via CIS Controls:

  • CIS 1–2: Asset inventory and discovery (hardware/software).
  • CIS 3: Data protection and encryption.
  • CIS 4–6: Secure configurations, access control, account management.
  • CIS 7–11: Vulnerability management, log auditing, malware defense, backups.

Tools Used:

  • Infrastructure & Automation: Kubernetes, Ansible, Jenkins, Docker.
  • Security & Monitoring: ClamAV, OCS Inventory, Nmap, Firewall/VPN.
  • Databases: MySQL (app data), PostgreSQL (logs).
  • Backup & Logging: Restic (backup), Grafana (dashboard).
  • CI/CD: GitLab, Jenkins pipelines.

What Did I Apply

  • DevOps practices: CI/CD with Jenkins and GitLab.
  • Infrastructure as Code: With Ansible and Kubernetes.
  • Security standards: CIS benchmarks and hardening via Ansible.
  • Scalable architecture: Using container orchestration (K8s), shared hosting, and backup solutions.
  • Monitoring and observability: With Grafana and log centralization.
  • Platform services management: Including automated vulnerability scanning and encrypted backups.
